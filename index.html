Puntuación de Fraude en Siniestros

Caso de Uso 01: Puntuación de Fraude en Siniestros

Contexto de Negocio — Sector Seguros

Este notebook demuestra la detección de fraude en siniestros mediante clasificación ML en Snowflake.

Foco: Siniestros de seguros (auto, hogar, vida) — detección de fraude en la declaración del siniestro.

Reto de negocio: Predecir y prevenir el fraude en siniestros antes del pago, reduciendo pérdidas hasta un 40%.

Objetivos de Aprendizaje

  • Crear flujos ML con SNOWFLAKE.ML.CLASSIFICATION
  • Generar datos sintéticos de siniestros de seguros
  • Construir variables predictoras a partir de datos del siniestro y del tomador
  • Evaluar modelos de clasificación binaria
  • Construir dashboards de puntuación de riesgo con Streamlit

Funcionalidades

  • Modelo ML: Clasificación binaria para predicción de fraude
  • Datos sintéticos: 500 tomadores, 1.000 siniestros con indicadores de fraude
  • Variables: dias_hasta_notificacion, importe_declarado, num_testigos, hora_accidente, taller_sospechoso, ratio_siniestros_ano
  • Dashboard Streamlit: Puntuación interactiva y filtrado por nivel de riesgo

Datos Generados

  • Tomadores: Datos demográficos, antigüedad, historial de siniestros previos
  • Siniestros: Fecha, importe, tiempo de notificación, testigos, taller
  • Predicciones ML: Nivel de riesgo (Alto/Medio/Bajo) con probabilidades

Estratificación de Riesgo

  • Alto Riesgo (≥70%): Derivar a equipo de investigación
  • Riesgo Medio (30-70%): Revisión por perito senior
  • Bajo Riesgo (<30%): Tramitación normal

Cómo Usar

  • Ejecutar las celdas secuencialmente de arriba a abajo
  • La configuración del entorno crea una base de datos aislada
  • Los datos sintéticos generan 1.000 siniestros automáticamente
  • El modelo se entrena en 30-60 segundos
  • El dashboard se lanza interactivamente en el notebook
