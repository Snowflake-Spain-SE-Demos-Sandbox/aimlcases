Detección de Anomalías IoT

Proyecto end-to-end de ML con Cortex Code en Snowflake
Smart Manufacturing · 50 máquinas · 100K lecturas · 8 prompts

1. El Problema

Una fábrica inteligente opera 50 máquinas equipadas con sensores IoT que registran temperatura, vibración, humedad, presión y consumo energético cada minuto. El dataset contiene 100.000 lecturas con un 8.9% de anomalías — fallos potenciales que, si no se detectan a tiempo, causan paradas no planificadas y pérdidas económicas.

El reto: construir un sistema completo de detección de anomalías — desde la ingesta de datos hasta la inferencia en producción — todo dentro de Snowflake, usando únicamente prompts en lenguaje natural.

2. Arquitectura General

Todo el proyecto vive dentro de Snowflake. El Compute Pool ejecuta el notebook con Container Runtime, las Network Rules permiten acceso a Kaggle y PyPI, y el modelo queda registrado para inferencia batch vía Warehouse.

3. Pipeline ML — Paso a Paso

Cada etapa del pipeline fue generada por Cortex Code a partir de un único prompt en español.

4. Qué es Cortex Code

Cortex Code es un asistente de IA integrado en Snowsight que entiende el contexto de tu cuenta Snowflake — roles, warehouses, bases de datos, tablas — y genera código directamente en tu workspace.

Decisiones inteligentes que tomó Cortex Code

  • Detectó data leakage: identificó que DOWNTIME_RISK (correlación 1.0) y MAINTENANCE_REQUIRED (correlación 0.63) no debían usarse como features
  • Group-based split: separó train/test por máquinas completas (1-40 vs 41-50), evitando fuga temporal
  • Class weighting: calculó scale_pos_weight = 10.29 para compensar el desbalance 91%/9%
  • Target WAREHOUSE: eligió la plataforma correcta para inferencia batch sin SPCS
  • Auto-corrección: cuando los gráficos no se mostraban, diagnosticó el backend de matplotlib y lo corrigió

7. Ventajas de Prompts en Lenguaje Natural

Método Tradicional — ~17 horas

  • Investigar docs de SPCS, Network Rules — 2h
  • Escribir SQL de infraestructura — 1h
  • Script de descarga + carga + debug — 2h
  • Código EDA + gráficos + análisis leakage — 4h
  • Feature engineering manual — 3h
  • Entrenar, evaluar, visualizar — 3h
  • Registrar en Model Registry + debug — 2h

Con Cortex Code — ~5 minutos

  • "Configura compute pool y external access" — 30s
  • "Descarga dataset y cárgalo en IOT" — 45s
  • "EDA completo con visualizaciones" — 1 min
  • "Construye features_df con rolling stats" — 30s
  • "Entrena XGBClassifier con class weighting" — 30s
  • "Evalúa rendimiento sobre test set" — 30s
  • "Registra modelo en Model Registry" — 30s

¿Por qué funciona?

1. Contexto persistente

Cortex Code recuerda toda la conversación. El prompt 5 dice "entrena" y sabe que features_df, train_df y X_train ya existen.

2. Conocimiento de Snowflake

Sabe que Container Runtime necesita Compute Pool, que write_pandas requiere session.connection, que WAREHOUSE evita SPCS.

3. Mejores prácticas de ML

Implementa group-based split, calcula scale_pos_weight, excluye columnas con leakage, usa PR-AUC (no solo accuracy).

4. Iteración rápida

Si algo falla (ej: gráficos no visibles), basta con decirlo en lenguaje natural. Cortex Code diagnostica y corrige automáticamente.

8. Uso en Producción

El modelo registrado con TargetPlatform.WAREHOUSE se puede invocar desde SQL o Python sin configurar contenedores.

Desde SQL

SELECT *
FROM COCO_AD.PUBLIC.IOT_ANOMALY_DETECTOR!PREDICT(
    SELECT TEMPERATURE, VIBRATION, HUMIDITY, PRESSURE,
           ENERGY_CONSUMPTION, MACHINE_STATUS,
           -- ... (las 24 features)
    FROM nueva_tabla_de_lecturas
);

Desde Python (cualquier notebook)

from snowflake.ml.registry import Registry

reg = Registry(session, database_name="COCO_AD", schema_name="PUBLIC")
mv = reg.get_model("IOT_ANOMALY_DETECTOR").version("V1")
predictions = mv.run(nuevo_dataframe, function_name="predict")
